# HearSpace 产品需求文档（PRD）

## 0. 文档状态

- 产品名称：HearSpace
- Slogan：Hear the Space.
- 当前版本：Product System 0.1
- 当前阶段：从 MVP 验证转向个人空间记忆产品化
- 当前核心形态：图片驱动的空间情绪生成、音乐记忆推荐与个人 Archive 系统
- 当前技术栈：Next.js / TypeScript / Tailwind CSS / Framer Motion / Qwen-VL via DashScope

一句话定义：

> AI 理解一张空间照片里的光线、场景与停顿，并把它生成一段可保存、可分享、可继续聆听的空间情绪记忆。

---

# 一、产品定位

## 1.1 产品本质

HearSpace 不是：

- 传统音乐播放器
- AI 文案工具
- 图片社交平台
- 数据分析工具
- 通用图片美化工具

HearSpace 的本质是：

> AI 空间情绪内容平台。

它通过图片理解、情绪语言、中文音乐记忆与个人归档，帮助用户记录“空间里的情绪瞬间”。

## 1.2 核心价值

传统社交媒体记录的是：

- 人
- 事件
- 生活展示
- 社交关系
- 人设表达

HearSpace 记录的是：

> 空间情绪（Spatial Emotion）。

典型空间包括：

- 深夜便利店
- 雨天公交站
- 飞机舷窗
- 校园走廊
- 酒店房间
- 公园长椅
- 窗边房间
- 空荡地铁

用户真正想保存的不是“我做了什么”，而是：

> 这一刻的氛围。

# 二、目标用户

## 2.1 核心用户：城市情绪记录者

年龄：18-30

特征：

- 喜欢拍空间、夜景、窗边、街角、房间、校园与城市细节
- 喜欢 citywalk、ambient、lofi、city pop、华语记忆型音乐
- 有耳机文化和私密听歌习惯
- 喜欢安静、慢节奏、低刺激内容
- 不希望每次记录都变成社交表演

典型平台语境：

- 小红书
- Instagram
- Threads
- 网易云音乐
- TikTok / 抖音

## 2.2 次级用户：情绪型内容创作者

需求：

- 用照片获得情绪文案灵感
- 自动匹配氛围音乐
- 快速生成适合社交平台发布的竖版情绪便签
- 保持内容高级、克制、有电影感

## 2.3 次级用户：轻度孤独型用户

需求：

> 记录某个瞬间，而不是展示自己。

# 三、当前版本产品目标

## 3.1 Product System 0.1 目标

当前阶段目标：

> 在“空间照片 → 情绪语言 → 中文音乐记忆 → 可保存分享”的闭环基础上，建立个人空间记忆盒的产品雏形。

核心体验路径：

1. 用户进入首页情绪流，感受产品气质。
2. 用户上传一张空间照片。
3. 用户可补充一句当下感受。
4. AI 结合图片与用户补充语境生成结构化空间情绪结果。
5. 系统匹配 3 首真实中文音乐记忆。
6. 用户查看沉浸式结果页。
7. 用户可下载 9:16 情绪便签。
8. 用户可保存到本地空间记忆归档。
9. 用户可在 Archive 中看到周/月回顾和相似记忆。

## 3.2 当前版本边界

当前版本暂不做：

- 短视频上传
- 实时摄像头分析
- 环境音识别
- 用户账号系统
- 公开社区发布
- 社交关注关系
- 真实地图定位
- AI 音乐生成
- Spotify / Apple Music 接入

这些能力进入后续阶段规划，不属于当前验收范围。

# 四、当前信息架构

## 4.1 首页：Mood Feed

路由：`/`

定位：

> 空间记忆电台，而不是工具入口页。

当前能力：

- 展示预置空间情绪样本
- 图片以电影感大画幅出现
- 支持上下切换空间记忆
- 支持键盘上下键与移动端滑动切换
- 配套本地 atmosphere audio
- 引导用户进入上传流程

产品要求：

- 首页第一屏必须先建立产品气质
- 不做营销式 landing page
- 不堆功能介绍
- 让用户先感到“这是什么样的情绪空间”

## 4.2 捕捉页：Capture Space

路由：`/capture`

当前能力：

- 上传一张图片
- 拖拽上传图片
- 本地预览图片
- 用户可补充一句当下感受，作为 AI 文案语境
- 前端压缩图片后提交
- 限制非图片文件和过大图片
- 生成过程中展示沉浸式 atmosphere loading
- 生成成功后进入结果页

核心交互原则：

- 极简
- 安静
- 不展示复杂 AI 分析过程
- 让用户感觉是在“保存一段空间”，而不是填写表单

## 4.3 结果页：Mood Result

路由：`/result`

当前能力：

- 从 `sessionStorage` 读取本次 MoodResult 与图片
- 以沉浸式电影画幅展示图片、时间标签、标题与副标题
- 展示空间记忆文案与空间人格描述
- 展示 3 首 Music Memory 推荐
- 支持替换单首推荐音乐
- 支持保存到本地空间记忆归档
- 支持预览并下载 9:16 情绪便签 PNG

结果页展示原则：

- 图片第一，文字第二，音乐第三
- 不以字段表格展示 AI 输出
- 不出现数据面板感
- 不把结果页做成“分析报告”
- 情绪文案像旁白，音乐像记忆锚点

## 4.4 归档页：Space Memory Archive

路由：

- `/archive`
- `/archive/[id]`

当前能力：

- 从 `localStorage` 读取用户保存的空间记忆
- 按日期分组展示
- 展示本周 / 本月空间记忆回顾
- 展示图片、标题、时间、记忆文字、音乐标签
- 支持进入单条详情
- 详情页展示用户补充的一句话
- 详情页展示相似空间记忆
- 支持删除归档记忆

当前数据存储：

- 本地浏览器 `localStorage`
- 无云端账号
- 无跨设备同步

后续可升级为用户账号与云端 archive。

# 五、AI 生成体系

## 5.1 当前输入

当前版本输入：

- 一张图片
- 用户补充的一句话（可选）
- 系统当前上下文
- 图片内可见空间线索

暂未接入：

- 环境音频
- 地理位置
- 天气
- 视频帧

## 5.2 当前模型

默认模型：

- Qwen-VL / Qwen2.5-VL

调用方式：

- 通过 DashScope API
- 环境变量：`DASHSCOPE_API_KEY`
- 可配置模型：`QWEN_VL_MODEL`

Mock 策略：

- 默认缺少 API Key 或调用失败时返回错误
- 只有 `HEARSPACE_ALLOW_MOCK=true` 时允许 mock fallback
- 前端会阻止 mock 结果伪装成真实图片分析结果

## 5.3 MoodResult 数据结构

AI 必须返回结构化结果，核心字段包括：

- `scene_observation`：空间观察，包括场景、物件、活动、光线、色调、拍摄风格、氛围证据
- `mood_title`：空间情绪标题
- `mood_subtitle`：副标题
- `time_label`：时间感标签
- `writing`：情绪长文案
- `space_memory_text`：主记忆文本
- `space_personality`：空间人格描述
- `visual_tone`：视觉气质标签
- `music_query`：音乐搜索语义
- `music_keywords`：音乐匹配关键词
- `music_memories`：AI 生成的音乐记忆描述
- `share_card_text`：便签短文案
- `visual_mood_tags`：展示型情绪标签
- `debug_source`：结果来源

结构化输出必须通过前端 schema 校验后才能进入结果页。

## 5.4 AI 输出原则

用户不应看到：

- arousal
- valence
- 恢复性指数
- 模型推理过程
- 学术术语
- 置信度面板

用户应该看到：

> 情绪语言。

文案风格：

- 真实人类日记感
- 电影旁白感
- 中文语境自然
- 有留白
- 不鸡汤
- 不模板化
- 不像 AI 总结

# 六、音乐记忆体系

## 6.1 产品定位

HearSpace 里的音乐不是背景音，也不是随机推荐。

音乐是：

> 这段空间记忆的声音锚点。

当前核心闭环：

> 空间照片 → 空间情绪 → 中文空间记忆类型 → 真实歌曲推荐 → 网易云继续聆听。

## 6.2 当前实现

当前版本采用：

- 本地 curated music library
- 音乐元数据缓存
- 空间记忆分类体系
- tag overlap scoring
- 网易云搜索/歌曲链接入口
- 结果页 Top 3 推荐
- 可替换单首推荐

当前不做：

- AI 生成歌曲
- 完整在线播放版权链路
- Spotify / Apple Music 接入
- 网易云非官方登录体系

## 6.3 空间记忆分类

当前分类包括：

- 校园 / 教室 / 青春
- 公园 / 草地 / 恢复性空间
- 深夜城市 / 餐厅 / 微醺
- 雨后 / 窗边 / 独处
- 东方庭院 / 园林 / 水面
- 花园 / 少女 / 柔光 / 梦幻
- 旅行 / 风景 / 远方
- 日常生活 / 房间 / 居住感
- 模糊但柔和的记忆

分类依据：

- 画面触发词
- 情绪词
- 空间场景
- 视觉气质
- 中文音乐文化语境

## 6.4 推荐结果要求

每首推荐至少需要：

- 歌曲名
- 艺术家
- 推荐理由
- 匹配信号
- 封面或占位图
- 网易云继续聆听入口

推荐理由要像“这首歌如何贴住这个空间”，而不是普通音乐 App 的标签解释或匹配报告。

## 6.5 Music System V2

V2 的目标不是单纯增加歌曲数量，而是建立可长期复用的数据资产：

> Space Memory Taxonomy × Music Knowledge Library。

### Space Memory Taxonomy

统一三级结构：

- Level 1：空间类型，例如 Natural Space、Urban Space、Residential Space、Transit Space、Campus Space、Travel Space、Commercial Space、Solitude Space
- Level 2：具体场景，例如 Classroom、Garden、Night Street、Train Window、Airport Waiting、Room、Restaurant
- Level 3：氛围标签，例如 Rainy、Sunset、Warm Light、Soft Focus、Quiet、Nostalgic、Dreamy、Lonely、Healing、Summer

所有歌曲、MoodResult、Archive 均复用该体系。

### Music Knowledge Library

每首歌必须逐步沉淀为知识库记录，核心字段包括：

- `spaceTags`
- `sceneTags`
- `emotionTags`
- `memoryTags`
- `visualTags`
- `seasonTags`
- `atmosphereTags`
- `similarSpaces`
- `recommendationReason`
- `confidence`

旧字段如 `memoryTypes`、`scenes`、`emotions` 保留作为兼容层和迁移依据。

### Weighted Space Memory Matching

推荐机制从简单 tag overlap 升级为加权空间记忆匹配：

1. `spaceTags`
2. `sceneTags`
3. `emotionTags`
4. `memoryTags`
5. `visualTags`

系统输出匹配得分，并用于排序、替换歌曲和后续 Feed / Community / Music Map 复用。

### Similar Spaces

每首歌拥有 `similarSpaces`：

> 这首歌也常出现在这些空间记忆里。

该字段用于增强推荐可信度，也会成为未来相似记忆、社区聚类和城市音乐地图的基础。

# 八、视觉与动效体系

## 8.1 产品气质

HearSpace 最终确认的产品方向：

> 深夜城市感 × 电影感空间日记。

关键词：

- 高级
- 克制
- 安静
- 有深度
- 情绪化
- 留白感
- 城市孤独感
- 空间叙事感

## 8.2 视觉语言

当前视觉体系参考：

- Apple minimal
- old film
- Wong Kar Wai warmth
- quiet archival memory

设计原则：

- 图片是第一主角
- 文字像电影字幕或私人日记
- 音乐推荐像 tape fragment
- 卡片只用于重复记忆、便签、局部模块
- 避免 dashboard、科技蓝、AI 助手感、功能堆叠

详见：

- `docs/hearspace-visual-language-system.md`
- `docs/hearspace-motion-design-system.md`

## 8.3 动效原则

动效不是装饰，而是记忆浮现的节奏。

当前动效语言：

- 慢速图片显影
- Ken Burns 漂移
- 胶片颗粒
- 暗角保护文字可读性
- 字幕式慢入场
- 滚动出现
- loading 过渡溶解

避免：

- 弹跳动画
- 快速 hover scale
- 彩带、徽章、游戏化反馈
- SaaS 卡片 hover lift
- 过度说明型 loading

# 九、产品原则

## 原则 1：空间是主角

产品表达重点是空间里的光、空气、停顿与情绪，不是用户本人。

## 原则 2：不解释 AI

AI 应该被隐藏。用户感受到的是：

> 它懂这个瞬间。

## 原则 3：情绪比功能重要

功能是为了形成记忆，不是为了展示技术能力。

## 原则 4：不做高刺激内容

HearSpace 必须保持慢节奏、低刺激、安静、沉浸。

## 原则 5：音乐必须有记忆感

推荐音乐要贴合中文语境、空间气质与画面线索，而不是泛化的“适合放松”。

## 原则 6：结果不是报告

所有 AI 输出都必须被转译成用户可感受的叙事、文字、音乐与视觉体验。

# 十、当前验收标准

## 10.1 核心链路验收

必须满足：

- 首页可正常展示 Mood Feed
- 用户可进入上传页
- 用户可上传图片并预览
- 用户可补充一句当下感受
- 图片提交到 `/api/generate-mood`
- 缺少 API Key 时明确报错，不返回伪真实结果
- 成功生成后进入结果页
- 结果页显示图片、标题、时间标签、文案、空间人格
- 结果页展示 3 首音乐推荐
- 用户可替换推荐歌曲
- 用户可预览并下载情绪便签
- 用户可保存记忆到 Archive
- Archive 可读取、展示、进入详情、删除
- Archive 可展示本周 / 本月回顾
- 详情页可展示相似记忆

## 10.2 内容验收

AI 结果必须：

- 有具体空间观察依据
- 标题不生硬
- 文案不模板化
- 音乐关键词能驱动推荐
- 音乐推荐理由自然、情绪化，不像匹配解释
- share card 文案适合短句展示
- 视觉标签简洁可读

## 10.3 体验验收

必须避免：

- 生成过程像后台任务
- 结果页像字段清单
- 音乐推荐像普通歌单
- 分享卡文字溢出
- 移动端布局重叠
- debug 信息在生产环境暴露

# 十三、长期护城河

HearSpace 的长期壁垒不是单点 UI、音乐列表或文案生成，而是：

> 空间 → 情绪 → 声音 → 记忆 的映射体系。

长期资产包括：

- 空间情绪标签体系
- 中文空间记忆分类
- 视觉线索与音乐语义映射
