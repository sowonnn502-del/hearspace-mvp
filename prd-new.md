# HearSpace 产品需求文档（PRD）

## 产品名称

HearSpace

Slogan：

> Hear the Space.

一句话定义：

> AI 理解空间氛围，并为这一刻生成属于它的情绪内容。

---

# 一、产品定位

## 1.1 产品本质

HearSpace 不是：

* 音乐播放器
* AI 文案工具
* 图片社交平台
* 视频剪辑工具

HearSpace 的本质是：

# AI 空间情绪内容平台

通过图片、声音、环境与情绪的联动，帮助用户记录“空间里的情绪瞬间”。

---

## 1.2 产品核心价值

传统社交媒体记录的是：

* 人
* 生活
* 社交关系
* 人设

HearSpace 记录的是：

# 空间情绪（Spatial Emotion）

例如：

* 深夜便利店
* 雨天公交站
* 飞机舷窗
* 酒店走廊
* 凌晨办公室
* 海边黄昏
* 空荡地铁

用户真正想记录的，并不是“我是谁”。

而是：

# “这一刻的氛围。”

---

### 抖音 / TikTok

# 十、AI 能力架构

## 第一阶段（MVP）

HearSpace 第一版采用：

# 图片优先的轻量多模态架构

第一版不接入环境音识别，不做实时摄像头分析，不做复杂视频理解。

核心输入：

* 图片
* 时间信息
* 地点信息（可选）
* 用户补充的一句话（可选）

核心输出：

* Mood Title
* 情绪长文案
* 空间人格
* 音乐推荐
* Mood Card
* Archive 归档

---

## 空间理解模型

MVP 阶段不自研 CV 模型，不训练复杂视觉模型。

视觉理解层采用可替换模型结构：

# Vision Model Adapter

默认模型：

# Qwen-VL / Qwen2.5-VL

备选模型：

* Kimi / Moonshot 多模态能力
* MiniMax 多模态能力

第一版明确采用：

# Qwen-VL / Qwen2.5-VL 作为默认视觉理解模型

Kimi / MiniMax 暂作为后续备选评测对象。

原因：

* Qwen2.5-VL 是 Qwen 官方发布的视觉语言模型，适合图片理解、图像描述和结构化视觉信息提取。
* Qwen2.5-VL 支持稳定 JSON / 结构化输出，适合 HearSpace 后续把图片分析结果传给文案生成、音乐匹配和 Mood Card 生成模块。
* Qwen2.5-VL 具备视觉定位、图像理解、文本识别和复杂视觉场景理解能力，能够覆盖 MVP 阶段的空间氛围识别需求。
* Kimi / Moonshot 在中文语义与长文本能力上具备优势，可作为文案润色模型备选。
* MiniMax 未来可作为音乐、视频、多模态扩展备选。

---

## AI 工作流

### Step 1：图片上传

用户上传一张空间图片。

---

### Step 2：空间氛围识别

视觉模型提取：

* 场景类型
* 室内 / 室外
* 光线
* 色温
* 天气感
* 时间感
* 空间开敞度
* 人与空间关系
* 人的行为状态
* 穿着与活动氛围
* 城市感 / 校园感 / 居住感

注意：

# 不做人脸识别，不强调具体人物身份。

---

### Step 3：时间感生成

AI 生成时间维度：

* 一天中的时间：清晨、黄昏、深夜、凌晨等
* 月份 / 季节：春天、夏末、冬夜、雨季等
* 年份：由系统时间或用户选择产生

不强行识别 Life Phase，不替用户定义人生阶段。

---

### Step 4：Mood Generation

根据空间氛围与时间感生成：

* Mood Title
* 情绪标签
* 空间人格
* 情绪长文案

文案风格采用：

# 真实人类日记感 × 电影旁白感

即：

* 有私人感
* 有电影感
* 有留白
* 不鸡汤
* 不过度煽情
* 不像 AI 总结

---

### Step 5：Music-Memory Matching

音乐不是背景，而是记忆锚点。

系统根据：

* 空间氛围
* 时间感
* 城市感
* 情绪关键词
* 中国本土音乐语境

匹配最适合这一刻的音乐。

第一版优先调用真实歌曲库，而非 AI 生成音乐。

# 十六、HearSpace Prompt Architecture（开发前确认版）

## 目标

HearSpace 的 Prompt Architecture 目标不是让 AI “描述图片”，而是让 AI：

# 把图片转译成空间记忆。

第一版核心链路：

# 图片 → 空间氛围理解 → 时间感判断 → 情绪命名 → 空间人格 → 文案生成 → 音乐关键词

---

## MVP 输入

第一版只支持：

* 图片上传
* 当前时间 / 用户选择时间
* 地点信息（可选）
* 用户补充一句话（可选）

不支持：

* 环境音
* 视频
* 实时摄像头
* 地图点位系统

---

## Qwen-VL 图片理解 Prompt

### System Prompt

你是 HearSpace 的空间氛围理解模型。你的任务不是普通图像描述，而是理解图片中的空间氛围、时间感、光线、人与空间关系，并输出结构化 JSON。

你需要遵守：

* 不识别人脸身份
* 不猜测具体个人隐私
* 不输出学术术语
* 不写“AI 分析结果”这类措辞
* 不夸张煽情
* 不替用户定义人生阶段
* 重点理解整体空间氛围

---

### User Prompt Template

请分析这张图片，并为 HearSpace 生成结构化空间氛围结果。

你需要识别：

1. 场景类型
2. 室内 / 室外
3. 一天中的时间感
4. 季节 / 月份感
5. 光线与色温
6. 空间密度
7. 人与空间关系
8. 人的行为与活动氛围
9. 城市感 / 校园感 / 居住感等空间气质
10. 适合匹配的音乐情绪关键词

请只输出 JSON，不要输出解释文字。

## Mood Generation Prompt

### System Prompt

你是 HearSpace 的情绪内容导演。你的任务是基于空间氛围 JSON，生成一段具有“真实人类日记感 × 电影旁白感”的空间记忆内容。

你需要遵守：

* 不要像 AI 总结
* 不要鸡汤
* 不要过度煽情
* 不要解释图片
* 不要使用学术词汇
* 不要替用户定义人生阶段
* 文案要克制、有留白、有空间感和时间感
* 中文表达自然，可以少量使用英文标题，但不要过度

---

### User Prompt Template

请根据以下空间氛围 JSON，为 HearSpace 生成一条 Mood 内容。

输出字段包括：

* mood_title：2-8 个字或中英混合短标题
* time_label：一天中的时间 + 季节/月 + 年份
* space_personality：一句空间人格化描述
* writing：2-5 行中文情绪文案
* music_memory_reason：为什么这类音乐适合这个空间
* music_keywords：音乐匹配关键词
* visual_mood_tags：用于前端展示的情绪标签

请只输出 JSON。

## 文案生成规则

HearSpace 文案必须像：

# 私人电影日记。

允许：

* 空间描写
* 时间感
* 轻微怀旧
* 情绪留白
* 中文本土记忆

避免：

* “你一定很难过”
* “这代表着人生新阶段”
* “让我们珍惜当下”
* “该空间具有……”
* “图中显示……”

---

## 音乐匹配规则

音乐不是背景音，而是记忆锚点。

第一版音乐匹配优先级：

1. 华语青春 OST
2. 华语城市感歌曲
3. Ambient / Piano / Cinematic
4. City Pop / Jazzhop
5. 东方氛围电子

匹配逻辑：

* 校园 / 教室 / 青春感 → 华语青春 OST、钢琴、慢速弦乐
* 雨夜 / 街道 / 霓虹 → 华语 city pop、jazzhop、slow electronic
* 机场 / 高铁 / 窗边 → ambient、cinematic、漂浮电子
* 公园 / 海边 / 黄昏 → piano、acoustic、soft ambient
* 深圳 / 科技园 / 夜景 → electronic drift、urban ambient
